Use word boundary regex for return-to-await replacement

FunctionMaker.create uses str.replace('return', 'return await')
when wrapping async callables. This blindly replaces any
occurrence of the substring "return", breaking identifiers like
return_value (which becomes return await_value) and strings
containing the word "return".

Switching to re.sub with \b word boundaries ensures only the
return keyword is replaced, leaving identifiers and string
contents intact.

Fixes #140
